P2000 Web User Interface Help

Overview

The P2000 ARS archives data and generates detailed reports from that archived data. It provides the reporting tools essential for reacting to alarms, transactions, and audit messages, as well as mustering and guard tour information. It also supports the long-term archiving of data that may be required for government and regulatory compliance.

The P2000 ARS is installed on a standalone Microsoft® Windows® Server 2008 system with a Microsoft SQL Server™ 2012, eliminating the impact to system performance of generating reports on a P2000 Server. Using a standalone system also allows system components, such as data storage and memory, to be easily upgraded.

A P2000 ARS gathers live data from one or more P2000 v3.12 Servers; it can generate reports from that data and archive the data for up to 30 years.

Note:   While a P2000 ARS can gather live data from multiple P2000 v3.12 Servers at one time, a P2000 v3.12 Server can send live data to only one P2000 ARS at a time.

A P2000 ARS can store database backups made from P2000 Servers (versions 3.12, 3.11, 3.10, and 3.8) and from a P2000 v3.12 ARS. Database backups can be restored by a P2000 ARS and reports generated from the restored data.

Note:   Reports can be generated against the from only one restored database backup at a time; restored databases cannot be combined.

The P2000 ARS provides a Web-based interface for running and scheduling reports. You can run and schedule reports from virtually anywhere using the P2000 Web UI.

The P2000 ARS uses Microsoft SQL Server Reporting Services (SSRS) to create reports, providing a wide range of report types and export formats. It also uses SQL Server Integration Services (SSIS) for data migration tasks.
