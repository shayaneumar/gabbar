Text Fields

When specifying criteria for a report, some fields are text fields; that is, fields where you enter whatever text you want (as opposed to drop-down fields where you can only make a selection from a list of available options).

Some text fields have a limited number of entries that make sense. For example, for the Site Name field you are going to enter an asterisk (*) if you want the report to show data from all sites or you're going to specify the name(s) of configured sites.

For other fields, however, a wider variety of options make sense. For example, the Description field on the Alarm Activity report could use virtually any text to describe an incident.

Note:   Text you enter in text fields when specifying criteria for a report are not case sensitive. That is, "John" is treated the same as "john".

The P2000 ARS supports a variety of options in text fields to help you get exactly the records you want in a report.

Note:   The following table uses the term "cardholder" for example purposes only.

Enter

Result

* (asterisk), the default

Matches all content. All records with content in the text field will be part of the report.

cardholder

Matches content with cardholder in it.

cardholder*

Matches all content starting with cardholder.

*cardholder*

Matches any content with cardholder in it. Note there are no spaces used.

* cardholder *

Matches content with cardholder in the middle. Note the location of the spaces.

cardholder badge

Matches content with either cardholder or badge in it.

Blank

Matches no content; no records will be part of the report.
