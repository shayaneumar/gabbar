Scheduling an Audit Trail Report

This procedure describes scheduling an Audit Trail report.

To schedule an Audit Trail report:

1.     Log on to the P2000 Web UI and navigate to the Investigate page.

2.     Locate the report you want to run, then click the Schedule button for that report.

The Schedule page for the report type you selected appears.

3.     Enter a Scheduled Report Description for the report you are scheduling.

This can help others understand what is in the report.

4.     Specify criteria or accept the default criteria for your report.

The criteria for a scheduled Audit Trail report are:

●      Language: Specify a language for the report.

●      User Name: Show records for all users (*) or just for the users you specify.

●      Site Name: Show records for all sites (*) or just for the sites you specify.

●      Partition Name: Show records for all partitions (*) or just the partitions you specify.

●      Audit Type: Show records for all audit types (<any>) or just the audit types you specify.

●      Audit Action: Show records for all audit actions (<any>) or just the audit actions you specify.

●      Item Name: Show records for all item names (*) or just for specified item names.

●      Date/Time Range: Specify a time frame for the report. Select Current Month, Current Week, Last 30 Days, Last 7 Days, Last Month, Last Week, Today, or Yesterday from the drop-down list.

●      Sort Order Option: Select the criteria by which the report data will be sorted: Date Time, Item Name, or User Name.

●      Sort Order Sequence: Select whether the report data will be sorted in Ascending or Descending order.

5.     Choose to generate your report as an Microsoft Excel file (XLS) or as an Adobe PDF file by selecting the appropriate button.

6.     Specify when you want your report run:

●      Select Daily to get a daily report; you’ll need to specify the time the report should be run, a date to start running the report in the Starting on field, and an optional end date in the Ending on field.

●      Select Weekly to get a report once a week; you’ll need to specify the day and time the report should be run, a date to start running the report in the Starting on field, and an optional end date in the Ending on field.

7.     Specify what you want to do with your reports:

●      If you want to save your report to a file share, select File Share, enter a valid path in the Location field, then enter a valid User Name and Password if these are needed to save the file to the specified location.

●      If you want to email your report, select Email, enter one or more email addresses in the Email address(es) field, enter text to go into the Subject of the message, and finally enter text in the Comment field to go into the body of the message.

8.     Click Schedule Report.

After the report is generated, it will be emailed or saved to the specified network share in the specified format.

Because this is a scheduled report, you don’t have the option of opening it right away; that feature is only for run once reports.

If you open a scheduled Audit Trail Report after it gets created, the front page includes the type of report, the report criteria, and the beginning of the data. The final page also shows the number of records in the report.
