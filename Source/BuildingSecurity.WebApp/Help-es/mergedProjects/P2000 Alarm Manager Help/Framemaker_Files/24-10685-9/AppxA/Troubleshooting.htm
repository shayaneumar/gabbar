P2000 Web User Interface Help

Troubleshooting

Before contacting Johnson Controls® Technical Support, use the following information to help you troubleshoot issues with the P2000 Alarm Manager. As a general rule, first verify that your computer matches or exceeds the system requirements listed in Minimum System Requirements. See also Browser Limitations.

Alarm Manager Troubleshooting

Issue

Resolution

Login page does not appear

If accessing the Web Server from a client, the client may not have access to the Web server over the network. Consider the following:

●      Verify that both the Web server and client have valid IP addresses.

●      Verify that you can ping the Web server from the client computer by IP address and name.

●      Consult with the local network administrator for assistance.

Trouble logging on

Verify you have correctly entered the User Name and Password. Both fields are case-sensitive.

If you have exceeded the maximum number of logon attempts, a system administrator must unlock your account before you can retry.

Unable to view alarms

Your user account must have proper permissions, as defined in P2000.

Unable to acknowledge alarm

Your user account must have proper permissions, as defined in P2000.

See Alarm Life Cycle for more information.

Unable to respond to alarm

Your user account must have proper permissions, as defined in P2000.

If the alarm status is currently Pending, Acknowledgement may be required before you can complete the alarm.

See Alarm Life Cycle for more information.

Unable to complete or remove alarm

Your user account must have proper permissions, as defined in P2000.

Acknowledgement may be required before you can complete the alarm.

A response may be required before you can complete the alarm.

The alarm must be in a completion state (for example, Secure, N/A, or an alarm from a remote site).

See Alarm Life Cycle for more information.

Unable to respond, complete, or remove multiple alarms at once

To perform an action that affects multiple alarms, each alarm you wish to modify must conform to the conditions required to perform the desired action. For example, if you select to complete three alarms at once, but only two are qualified to be completed, you will only be able to complete those two alarms.

Sound does not turn off; plays continuously

The alarm status is still Pending. Once the status changes, the sound is no longer played. You can acknowledge the alarm to stop the sound from playing.
