Alarm Details and History

The P2000 Alarm Manager enables you to view the alarm details for each alarm, so that you have a complete picture of the alarm and a detailed history of previous actions associated with the alarm.

For step-by-step instructions on viewing alarm details, see Viewing Alarms and Alarm Details.

Alarm details are divided into two categories:

●      Alarm Specifics

●      Alarm History

The Alarm Specifics section enables you to:

●      View the P2000 location (site) where the alarm originated

●      See whether the alarm can be viewed across multiple P2000 partitions (public)

●      Identify the alarm category to which the alarm belongs, as defined in P2000

●      Determine the current alarm state (for example, Secure, N/A, and so on)

●      Identify whether the alarm has been escalated. For more information, see Alarm Escalation.

●      Read the associated alarm instructions, so you can act accordingly

For more information on the previous Alarm Specifics described, refer to the P2000 Software User Manual (Part No. 24-10685-157).

The Alarm History section enables you to view the following alarm details:

●      The date and time when the alarm action occurred (Action Date/Time)

●      The change in status as a result of the alarm action (Alarm Status). For example, if the current status is Pending and you enter a response, the alarm history displays the alarm status as Responding for that action item.

●      The user name of the operator who performed the action.

●      The change to the alarm state as a result of the alarm action.

●      The response text entered by the operator, if applicable.
