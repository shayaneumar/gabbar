Changing the Time Zone Setting

The P2000 Alarm Manager records the date and time each alarm occurs and displays it according to your local time zone settings in the Alarm Manager application. For example, if the Web server is located at a site that follows Central Time (UTC-06:00), and your Alarm Manager time zone settings are set to Pacific Time (UTC-08:00), the alarms that occurred at 10:00 PM Central Time appear as 8:00 PM when viewed on the client computer configured for Pacific Time. See P2000 Alarm Manager Time Zones Settings for other examples.

Note:   The default P2000 Alarm Manager time zone setting, regardless of the local operating system’s time zone setting, matches the Web server’s time zone setting. For example, if you connect from a client computer set for Pacific Time to a Web server set for Central Time, the client Alarm Manager displays the alarms, by default, in Central Time.

You can change the P2000 Alarm Manager time zone setting, if desired.

To change the time zone setting:

1.     Click the Settings button. The Time Zone drop-down list appears.

2.     Select a new time zone from the drop-down list.

The Alarm Manager page refreshes to display the alarms with updated time stamps based on the new time zone selected.
