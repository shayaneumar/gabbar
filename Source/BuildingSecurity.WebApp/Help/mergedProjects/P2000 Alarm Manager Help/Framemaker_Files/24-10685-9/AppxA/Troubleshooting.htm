P2000 Web User Interface Help

Troubleshooting

Before contacting Johnson Controls® Technical Support, use the following information to help you troubleshoot issues with the P2000 Alarm Manager. As a general rule, first verify that you are using a supported browser (refer to the Supported Web Browsers for the P2000 Web User Interface Technical Bulletin (LIT-12011833).

Alarm Manager Troubleshooting

Issue

Resolution

Sign In page does not appear

If accessing the web server from a client, the client may not have access to the web server over the network. Consider the following:

●      Verify that both the web server and client have valid IP addresses.

●      Verify that you can ping the web server from the client computer by IP address and name.

●      Consult with the local network administrator for assistance.

Trouble signing in

Verify you have correctly entered the User Name and Password. The password is case sensitive.

If you have exceeded the maximum number of sign in attempts, a system administrator must unlock your account before you can retry.

Unable to view alarms

Your user account must have proper permissions, as defined in P2000.

Unable to acknowledge alarm

Your user account must have proper permissions, as defined in P2000, and the alarm must have a status of Pending.

See Alarm Life Cycle for more information.

Unable to respond to alarm

Your user account must have proper permissions, as defined in P2000, and if the alarm status is currently Complete, you can no longer add a response. You can only respond to alarms with a status of Pending, Acknowledged, or Responding.

See Alarm Life Cycle for more information.

Unable to complete or remove alarm

Your user account must have proper permissions, as defined in P2000.

Acknowledgement may be required before you can complete or remove the alarm.

A response may be required before you can complete or remove the alarm.

The alarm must be in a completion state (for example, Secure, N/A, or an alarm from a remote site).

See Alarm Life Cycle for more information.

Sound does not turn off; plays continuously

The alarm status is still Pending. Once the status changes, the sound is no longer played. You can acknowledge the alarm to stop the sound from playing.
