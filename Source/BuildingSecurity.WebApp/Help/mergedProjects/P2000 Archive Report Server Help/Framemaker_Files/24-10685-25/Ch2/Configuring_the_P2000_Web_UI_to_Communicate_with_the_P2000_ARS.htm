Configuring the P2000 Web UI to Communicate with the P2000 ARS

To run or schedule a report using the P2000 Web UI, you must configure it to communicate with SSRS on the P2000 ARS. This is only required once, as long as the information stays the same.

Note:   The fields on the Report Server screen require knowledge of P2000 Web UI and P2000 ARS network settings. If you have any difficulty configuring communications between the two, contact your P2000 administrator or IT staff.

To configure communication between the P2000 Web UI and the P2000 ARS:

1.     Using a supported Web browser, log on to the P2000 Web UI.

2.     Click System Setup at the top of the screen, then slide down and select Report Server.

The Report Server screen appears.

3.     Enter the following information on the Report Server screen:

●      Use SSL/TLS check-box: Check to create an encrypted tunnel between the P2000 Web UI and the P2000 ARS using SSL/TLS. If you check this box, the Web Service URL and User Interface URL fields on this screen changes from http to https.

Note:   SSL must be configured in SSRS on the P2000 ARS for this feature to work. Johnson Controls recommends using SSL for added security.

●      Server (IP Address or Host Name): Enter the IP address or host name of the system hosting the P2000 ARS. The information in this field is used to create part of the Web Service URL field and the User Interface URL field on this screen.

●      Web Service Virtual Directory: Enter the location of the SSRS Web Service directory on the P2000 ARS if it has been changed from the default setting, which is ReportServer. The information in this field is used to create part of the Web Service URL field on this screen.

●      Web Service URL: This is the URL of the SSRS Web service running on the P2000 ARS. The default is: http://[IP address or machine name]/ReportServer. This field is not editable.

●      Domain: Only required if the account being used to access the P2000 ARS from the P2000 Web UI is an Active Directory account rather than a local machine account on the P2000 ARS. If necessary, consult your P2000 ARS administrator for more information.

●      User Name: This is the user name of a user account on the P2000 ARS that is authorized to run and schedule reports.

●      Password: This is the password of the user account. This field is case sensitive.

4.     Click Test Configuration to make sure the configuration information you entered is accurate. A status message appears under the Test Configuration button to indicate success or failure of the test of the current configuration.

If the current configuration is not correct, verify all of the settings and then test the configuration again.

5.     Click Save to save your configuration settings.
