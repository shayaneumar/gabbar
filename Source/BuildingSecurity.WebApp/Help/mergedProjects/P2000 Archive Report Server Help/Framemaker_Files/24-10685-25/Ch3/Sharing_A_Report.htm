Sharing A Report

When you schedule a report, you can either save it to a network file share or you can email it to one or more recipients:

●      Saving to a network file share: Enter a path to the file share location and, if required, a valid User Name and Password. The report file is saved to the specified location.

The user account that is scheduling the report must have write access permission to the file share location and users who want access to the report must have read permissions at that location. If you are having issues either saving to the file share location or reading from a file share location, contact your P2000 administrator or IT staff, or refer to the SSRS documentation from Microsoft.

●      Emailing: Enter one or more email addresses (separate addresses using a semicolon), and, if desired, a Subject for the subject line of the email and a Comment for the body of the message. The report file is an attachment to the message.

To email reports, you need an SMTP server or gateway, a user account that has permission to send email from the SMTP server or gateway, and SSRS must have Send As permissions on the SMTP server or gateway. If you are having issues either sending or receiving a report, contact your P2000 administrator or IT staff, or refer to the SSRS documentation from Microsoft or the documentation for your SMTP server or gateway.

Pick the desired sharing method when you create the scheduled report.
