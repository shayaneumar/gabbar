Running a Report

Running a report means to run it immediately with the desired criteria.

Note:   If you run a report that takes more than four minutes to create, the report will not be created; it will time out. If you run into this issue, you can schedule the report instead of run it or change the parameters so that less data is returned. Also, if you are creating a PDF, you can switch to CSV output; CSV files generally take less time to create.

To run a report:

1.     Log on to the P2000 Web UI, pull down the Reporting menu, and select Run a Report.

2.     On the Run a Report screen, select Standard or Custom, then click the name of the report you want to run.

The [report name] page for the report you selected appears.

3.     Specify criteria or accept the default criteria for the configuration of your report in the How do you want to configure your report? field.

Note:   Each report only uses a subset of the available criteria. Also, some criteria appear dimmed if they do not apply to the current selection.

Choose from the available criteria (see Configuration Criteria for a list of criteria) for the report you are running.

4.     Choose to generate your report as a PDF, a Microsoft Excel (XLS), or CSV file by selecting the appropriate button in the How do you want to generate your report? field.

5.     Specify a data source from the drop-down list in the What data source would you like to use? field:

●      Default means the live data being gathered by the P2000 ARS from one or more P2000 v3.12 Servers.

●      A named report refers to a report data backup that has been restored by the P2000 ARS.

Note:   This field displays only when there is more than one data source.

6.     Click Run Report.

After the report is generated and automatically saved, you are prompted to open it with an appropriate application.

If the system you are on cannot open the file (an appropriate application is not installed), you can open it at a later time.

7.     Open the report file if you would like to view it.
