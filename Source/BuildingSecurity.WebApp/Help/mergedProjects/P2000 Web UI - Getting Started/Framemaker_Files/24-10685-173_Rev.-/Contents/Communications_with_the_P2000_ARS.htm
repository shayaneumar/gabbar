Communications with the P2000 ARS

You can secure communication between the P2000 Web UI and the P2000 Archive Report Server (ARS) using HyperText Transfer Protocol Secure (HTTPS). The default communications mode is unsecured HTTP.

To establish HTTPS, use a self-signed SSL certificate generated by SQL Server Reporting Services (SSRS) on the P2000 ARS.

You must install the client portion of this SSL certificate on the P2000 Web UI.

To install the client portion of the SSL certificate on P2000 Web UI:

1.     Locate and double-click the certificate file. The Certificate dialog appears.

2.     Click Install Certificate. The Certificate Import Wizard appears.

3.     Click Next.

4.     Select Place all certificates in the following store, click Browse, select Show physical stores, expand Trusted Root Certification Authorities, select Local Computer, and click OK.

5.     Click Next, then click Finish.

6.     Click OK on the success dialog. The Certificate dialog appears.

7.     Note the text in the Issued to field. This information must match what you enter in the Server (IP Address or Host Name) field when you are configuring SSL/TLS in the P2000 Web UI.

8.     Click OK.

Once the certificate is installed, select Use SSL/TLS in the Web UI’s System Setup for Report Server. See P2000 Archive Report Server User Guide for details.
